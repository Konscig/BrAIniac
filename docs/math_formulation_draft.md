# Математическая постановка задачи построения агента

---

## Определение ноды

Каждая нода v_i ∈ V_all определяется:

- **Шаблон конфигурации (Ω_i)** — набор параметров, которые необходимо заполнить:
  - Ω_i = {p_1, p_2, ..., p_m}, где каждый p_j — параметр с именем, типом и областью допустимых значений
  - Примеры параметров: системный промпт (текст), список подключаемых нод (подмножество V_all), параметры модели (числовые), режим работы (перечисление) и т.д.
- **Конфигурация (C_i)** — конкретные значения всех параметров шаблона. C_i валидна, если каждое значение попадает в область допустимых значений своего параметра
- **Формат входа (In_i)** — схема данных, которую нода принимает
- **Формат выхода (Out_i)** — схема данных, которую нода возвращает
- **Функция преобразования:** O_i = f_i(I_i, C_i) — нода превращает вход в выход, где I_i соответствует In_i, а O_i соответствует Out_i
- **Вложенность:** нода может содержать дочерний граф G_child = (V_child, E_child), составленный из тех же V_all. В этом случае:
  - вход материнской ноды → вход дочернего графа
  - выход дочернего графа → выход материнской ноды
  - дочерний граф подчиняется тем же ограничениям, что и материнский (рекурсивно)
  - **порядок проверки:** сначала оцениваются дочерние агенты, затем материнский (снизу вверх)

---

## Дано

- **V_all = {v_1, ..., v_N}** — конечный известный каталог доступных нод, каждая со своим шаблоном Ω_i
- **D_gold = {(x_k, y_k)}** — Golden Dataset: набор пар (вход, эталонный выход), разделяется на:
  - **D_run ⊆ D_gold** (~20-30%) — для проверки запускаемости (задача построения)
  - **D_eval = D_gold \ D_run** (~70-80%) — для оценки качества (задача оценки)

## Найти

1. **V\* ⊆ V_all** — подмножество выбранных нод
2. **E\* ⊆ V\* × V\*** — связи между ними, такие что G\* = (V\*, E\*) — допустимый DAG
3. **C_i** для каждой v_i ∈ V\* — заполненный шаблон конфигурации (все параметры Ω_i заполнены валидными значениями)

Итого: тройка **(V\*, E\*, C)**, где C = {C_i | v_i ∈ V\*}.

## Ограничения

### Критерий запускаемости (через Golden Dataset)

**Runnable(G\*, C, D_run) = true** тогда и только тогда, когда:

1. G\* = (V\*, E\*) — связный DAG с единственным входом и выходом, где каждая нода достижима от входа и из неё достижим выход
2. Для каждого ребра (v_i, v_j) ∈ E\*: Out_i совместим с In_j
3. **Запрет обратных ссылок:** если нода v_i содержит дочерний граф G_child, то ни одна нода из G_child не может ссылаться на ноды материнского графа (ацикличность по иерархии)
4. **Для всех пар (x_k, y_k) из D_run агент:**
   - не падает (завершается без ошибок)
   - возвращает выход O, соответствующий Out_n финальной ноды графа

Пункты 1–3 — структурная проверка (статическая).
Пункт 4 — функциональная проверка (динамическая, через прогон на D_run).

## Результат

Тройка (V\*, E\*, C), удовлетворяющая всем ограничениям — **работающий агент**, готовый к передаче в задачу оценки.

---

## Связь с задачей оценки

Задача построения **предшествует** задаче оценки и является её необходимым условием. Без работающего агента оценка невозможна. Подробная постановка задачи оценки — см. [math_evaluation_draft.md](math_evaluation_draft.md).

## Закрытые вопросы

- [x] Шаблон конфигурации — набор параметров с именами, типами и допустимыми значениями; задача — заполнить валидно
- [x] Типы нод убраны — нода это просто элемент каталога V_all со своим шаблоном
- [x] R убрано — ресурсы (тулы, API) это тоже ноды из V_all
- [x] Критерий запускаемости — агент не падает и возвращает валидный формат на D_run (20-30% от D_gold)
- [x] Целевая функция — пока **не нужна**, задача feasibility (найти допустимое решение)
- [x] Эксплуатационные ограничения — убраны из построения, относятся к задаче оценки
- [x] Вложенность — дочерний граф из тех же V_all + запрет обратных ссылок + оценка снизу вверх
- [x] Входы/выходы — In_i и Out_i в определении ноды, совместимость через Out_i ~ In_j
- [x] Валидный формат — определён как соответствие Out_n финальной ноды
