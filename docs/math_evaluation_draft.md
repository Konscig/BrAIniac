# Математическая постановка задачи оценки агента

---

## Дано

- **(G\*, C)** — работающий агент (результат задачи построения)
- **D_gold = {(x_k, y_k)}** — Golden Dataset
- **Judge** — LLM-as-a-Judge (внешняя модель-оценщик)

## Переход от построения к оценке

Из работающей конфигурации (G\*, C) выводится множество применимых метрик, которое делится на два уровня:

**E(G\*, C) = E_node(G\*, C) ∪ E_agent(G\*, C)**

где:

- **E_node(G\*, C) = ⋃ E_node(T_i) для всех v_i ∈ V\*** — метрики уровня нод, определяются типом ноды
- **E_agent(G\*, C)** — метрики уровня агента, оценивают пайплайн целиком

### Метрики уровня ноды — E_node(T_i)

Зависят от типа ноды. Примеры:
- **Agent-нода:** качество ответа, адекватность, следование промпту
- **Tool-нода:** корректность вызова, точность результата
- **Logic-нода:** правильность ветвления
- **DataSource-нода:** полнота и корректность извлечённых данных

### Метрики уровня агента — E_agent(G\*, C)

Оценивают пайплайн как единое целое:
- Общее количество ошибок в пайплайне
- Метрики вызова тулов (сколько вызвано, сколько успешно)
- Сквозное время выполнения
- Сквозная стоимость
- Качество финального выхода (end-to-end)

## Объекты оценки

Оценка проводится на двух уровнях:

**Уровень ноды** — оценивается кортеж для каждой ноды:

**< v_i, C_i, O_i >**

где O_i — фактический выход ноды при прогоне на D_gold.

**Уровень агента** — оценивается агент целиком:

**< G\*, C, O_final >**

где O_final — финальный выход всего пайплайна.

## Найти

**Уровень ноды:** для каждой v_i ∈ V\*:

**score(v_i) = Judge(O_i, y_k, E_node(T_i))**

**Уровень агента:**

**score(A) = Eval(G\*, C, D_gold, E_agent)**

Часть метрик агента вычисляется автоматически (ошибки, вызовы, время, стоимость), часть — через Judge (качество финального выхода).

## Критерий качества

Агент считается качественным, если выполняются **оба** условия:

1. **По нодам:** для всех v_i ∈ V\*: score(v_i) > α_node
2. **По агенту:** score(A) > α_agent

где α_node и α_agent — пороговые значения допустимого качества.

## Поиск слабого звена

Если агент не проходит критерий качества:
- **Слабая нода:** argmin score(v_i) по всем v_i ∈ V\*
- **Проблема на уровне агента:** конкретная метрика из E_agent, не прошедшая порог

## Связь двух задач

```
Задача построения                    Задача оценки

V_all, R, D_gold                     (G*, C) — работающий агент
      |                                    |
      v                                    v
Найти (G*, C)  ──────────────>   Вывести E(G*, C)
      |                            /               \
  Runnable = true            E_node(T_i)       E_agent(G*,C)
  Эксплуатац. ок                |                   |
                                v                   v
                         score(v_i)            score(A)
                         по каждой ноде        по агенту
                                \                  /
                                 v                v
                          score(v_i) > α_node  И  score(A) > α_agent ?
                                       |
                                Да: агент годен
                                Нет: найти слабое звено
                                  - нода: argmin score(v_i)
                                  - агент: какая метрика < порога
```

Задача построения — **предшествует** задаче оценки и является её необходимым условием. Без работающего агента оценка невозможна. Постановка построения — см. [math_formulation_draft.md](math_formulation_draft.md).
