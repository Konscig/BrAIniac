# Глоссарий Предметной Области

## Назначение
Документ фиксирует единые термины для архитектуры BrAIniac, валидации и реализации.

## Термины
- Агент: поведение на этапе исполнения, которое может планировать шаги, вызывать инструменты и формировать итоговый результат.
- AgentCall-нода: специализированный тип узла (роль: transform), который выполняет агентный runtime внутри одной вершины графа.
- Инструмент (Tool): исполняемая возможность с контрактом входа/выхода и runtime-реализацией.
- Дефолты семейства инструментов: лимиты и политики по умолчанию, хранимые в Tool.config_json.
- NodeType: шаблон, определяющий поведение и ограничения узлов данного типа.
- Профиль NodeType: машиночитаемые ограничения, хранимые в NodeType.config_json.
- Узел (Node): конкретный экземпляр на канве, ссылающийся на один NodeType.
- Ребро (Edge): направленная связь между двумя узлами в рамках одного pipeline.
- Граф pipeline: направленный граф, построенный из сущностей Node и Edge.
- Строгий DAG: направленный ациклический граф pipeline.
- Preflight: полная проверка графа перед запуском.
- Hard-правило: нарушение блокирует запись или запуск.
- Soft-правило: нарушение возвращает только предупреждение.

## Двухуровневая Модель Согласованности
- Уровень 1 (модель данных / канва): Node и Edge образуют строгий DAG.
- Уровень 2 (runtime узла): AgentCall может выполнять внутренний bounded-цикл и tool calls.
- Внутренние вызовы инструментов не создают записи в Edge и не меняют топологию внешнего графа.

## Нормативная Лексика
- MUST: обязательное требование.
- SHOULD: рекомендуемое поведение по умолчанию.
- MAY: опциональное поведение.
