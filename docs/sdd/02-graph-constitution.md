# Конституция Графа

Constitution-Version: v2.1
Status: Draft

## Область Применения
Данная конституция регулирует построение и валидацию графов pipeline.

## Базовые Принципы
- P1: Внешний граф pipeline MAY содержать циклы.
- P2: Допускается несколько стартовых узлов.
- P3: Корректность графа MUST проверяться при изменении графа, а не только перед запуском.
- P4: Внутренняя логика агента выполняется локально внутри AgentCall и MUST NOT создавать ребра графа.
- P5: Обязательным ограничением для циклов MUST быть лимит maxIterations в JSON-профиле.

## Hard-Правила
- H1: Cross-pipeline edge запрещен.
- H2: Дублирование пары ребер (from,to) запрещено.
- H3: Любой цикл MUST иметь loop-policy с maxIterations >= 1 в JSON-профиле.
- H4: Профиль NodeType MUST быть разрешим (или применяется явная fallback-политика).
- H5: Перед мутацией MUST проходить ownership и auth-проверки.

## Soft-Правила
- S1: Проверка orphan-узлов SHOULD возвращать предупреждения.
- S2: Пороговые ограничения сложности SHOULD возвращать предупреждения (число узлов, глубина, ветвистость, число циклов).
- S3: Role-cardinality и role-compatibility SHOULD быть режимами warn/strict, а не обязательным hard-ограничением.
- S4: Отсутствующие необязательные поля профиля SHOULD возвращать предупреждения с применением дефолтов.
- S5: Глобальные бюджеты исполнения SHOULD быть включены в production-режиме.

## Принудительная Проверка При Мутациях
Следующие операции MUST выполнять hard-валидацию:
- create node
- update node
- delete node
- create edge
- delete edge

## Preflight
- Preflight MUST выполнять полную проверку графа и возвращать hard-ошибки и soft-предупреждения.
- Preflight MUST различать guarded и unguarded циклы.
- Preflight MAY включать предупреждения по рискам стоимости/времени.

## Политика Типов Нод
- Типы нод являются в первую очередь семантическими метками и шаблонами конфигурации.
- Циклы допускаются между любыми loop-capable NodeType (включая transform и tool-like ноды).
- Жесткое требование к типам нод: наличие корректного maxIterations для циклических маршрутов.

## Процесс Изменения Конституции
1. Обновить текущую конституцию.
2. Обновить профили ролей и каталог ошибок.
3. Обновить API-контракт и тесты.
4. Выпустить новую версию с повышением version.
