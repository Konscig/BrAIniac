# Конституция Графа

Constitution-Version: v1.0
Status: Draft

## Область Применения
Данная конституция регулирует построение и валидацию графов pipeline.

## Базовые Принципы
- P1: Внешний граф pipeline MUST быть строгим DAG.
- P2: Допускается несколько стартовых узлов.
- P3: Корректность графа MUST проверяться при изменении графа, а не только перед запуском.
- P4: Внутренняя логика агента выполняется локально внутри AgentCall и MUST NOT создавать ребра графа.

## Hard-Правила
- H1: Self-loop запрещен.
- H2: Cross-pipeline edge запрещен.
- H3: Дублирование пары ребер (from,to) запрещено.
- H4: Роль source MUST иметь ноль входящих ребер.
- H5: Роль sink MUST иметь ноль исходящих ребер.
- H6: Ограничения кардинальностей входа/выхода по роли MUST выполняться.
- H7: Совместимость ролей (predecessor/successor) MUST выполняться.
- H8: После каждого изменения узла или ребра граф MUST оставаться ацикличным.
- H9: Профиль NodeType MUST быть разрешим (или применяется явная fallback-политика).
- H10: Перед мутацией MUST проходить ownership и auth-проверки.

## Soft-Правила
- S1: Проверка orphan-узлов SHOULD возвращать предупреждения.
- S2: Пороговые ограничения сложности SHOULD возвращать предупреждения (число узлов, глубина, ветвистость).
- S3: Отсутствующие необязательные поля профиля SHOULD возвращать предупреждения с применением дефолтов.

## Принудительная Проверка При Мутациях
Следующие операции MUST выполнять hard-валидацию:
- create node
- update node
- delete node
- create edge
- delete edge

## Preflight
- Preflight MUST выполнять полную проверку графа и возвращать hard-ошибки и soft-предупреждения.
- Preflight MAY включать предупреждения по рискам стоимости/времени.

## Процесс Изменения Конституции
1. Обновить текущую конституцию.
2. Обновить профили ролей и каталог ошибок.
3. Обновить API-контракт и тесты.
4. Выпустить новую версию с повышением version.
