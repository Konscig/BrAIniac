# Заметки По Миграции: NodeType.config_json

## Цель
Добавить машиночитаемое хранение профилей в NodeType с минимальным изменением схемы.

## Планируемое Изменение
- Добавить NodeType.config_json как поле JSON/JsonB.
- Сохранить Tool.config_json для дефолтов семейства инструментов.

## Рекомендуемая Форма Prisma
```prisma
model NodeType {
  type_id      Int    @id @default(autoincrement())
  fk_tool_id   Int
  name         String @db.Char(64)
  desc         String @db.Char(512)
  config_json  Json?
  // relations...
}
```

## Стратегия Заполнения
1. Прочитать дефолты из Tool.config_json.
2. Сформировать профиль NodeType для каждого типа.
3. Записать данные в NodeType.config_json.
4. Сохранить fallback-поведение на переходный период.

## План Внедрения
1. Применить миграцию.
2. Выкатить код, который сначала читает NodeType.config_json, затем дефолты из Tool.
3. Запустить скрипт заполнения профилей.
4. Включить strict-режим после полного покрытия.

## Критерии Приемки
- Миграция применяется без ошибок.
- Существующие NodeType остаются работоспособными.
- Валидатор разрешает профили для всех активных NodeType.
- Интеграционные тесты проходят в strict-режиме.
